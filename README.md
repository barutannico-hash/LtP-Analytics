# 初心者大会 戦績分析ダッシュボード（再設計版）

参考記事の手法をベースに、初心者大会の**指導・成長支援**向けに再設計した戦績分析ツールです。
Riot API から各選手の戦績＋タイムラインを収集し、**4ページ構成のインタラクティブ HTML ダッシュボード**を生成します。

- ① 大会全体の概要 ② 選手一覧＆比較 ③ 選手詳細（指導の核） ④ ロール別ベンチマーク
- ⑤ チャンピオンプール（習熟度）

### ドラフト準備機能（⑤）
- **⑤ チャンピオンプール（習熟度）**：SHGアナリストレポート風に、選手ごとに **得意/普通/練習中** をアイコンのマトリクスで表示。
  戦績から自動分類（5戦以上=得意 / 2〜4戦=普通 / 1戦=練習中）し、アイコンのクリックで習熟度を切替・×で除外・任意のチャンプを追加できます（変更はブラウザに保存）。
  各選手の下に**チャンピオン別戦績テーブル（使用回数・勝率・KDA・CS/min等）**を表示。**コーチのチャンピオンプール／戦績もおまけで表示**します。
- **④ ロール別ベンチマーク**：ロール内の選手＋各コーチを実数値で比較する表と、指標別ランキングを表示（レーダーチャート・相関図は削除）。
- **対AI戦（Co-op vs AI）は集計から除外**しています。

### 最新の追加機能（UI改善）
- **指定レーンのみ集計**：`players.csv` で各選手に指定したロールと、実際に担当したロール（teamPosition）が一致する試合だけを反映。
  指定外のレーンや、ポジション情報の無い試合（ARAM等）は除外します（コーチは担当レーン指定が無いためベンチマークとして全試合を保持）。
- **選手×ロール別の戦績**：各試合の実際の担当ロール（teamPosition）を取得し、選手ごとに**ロール別で集計**。
  選手詳細・選手一覧・ロール別ベンチマークでロールを切り替えると、試合数・戦績・チャンピオンプールがそのロールのものに変わります。
- **チャンピオンアイコン表示**：チャンプ名の横にアイコン画像（Data Dragon）。選手詳細の各表・選手一覧の使用上位に反映。
- **選手別戦績テーブルをU.GG風に**：順位／アイコン＋名前／勝率(W-L)／KDA(K/D/A内訳)／最大K／最大D／CS／ダメージ／ゴールド。
- **各平均の説明を追加**：移動平均(直近5試合のトレンド)／ロール平均(同ロール選手の平均)を成長グラフ等に1行で明記。

### 以前の追加機能
- **チャンピオン名を日本語表示**（Data Dragon から最新の日本語名を自動取得。オフライン時は同梱の簡易辞書）。
- **総試合数は「ユニーク試合数」**（同じ試合の重複を除外）で表示。のべ出場数は小さく併記。
- **改善提案エンジン**：各選手の数値を **ブロンズ〜シルバー帯の基準**と比較し、選手詳細ページに
  「次に改善したいポイント」を上位3つ＋一言アドバイスで表示。基準値は `benchmarks.json` で自由に編集可能。
- **コーチとの比較を実数値テーブルに変更**（グラフではなく数字で差を表示。選手詳細＆ロール別ベンチマークに掲載）。

## いちばん簡単な使い方

設定が済んでいれば、**`update.bat` をダブルクリック**するだけです。
「最新の戦績＋タイムラインを取得 → 集計 → `output/dashboard.html` を開く」までを自動で行います。
（コマンドなら `python update.py`）

---

## ⚠️ 重要：タイムライン取得について（今回の追加）

CS@10・ゴールド差@10・レベル差@10・デスのタイミング分布・ファーストブラッド関与といった
**序盤(レーニング)の指標は、Riot の「Match Timeline API」が必須**です。今回これを収集に追加しました。

- すでに集めてある試合データには**タイムラインがまだありません**。
  → もう一度 `update.bat`（または `python collect.py`）を実行すると、**既存の試合のタイムラインだけを追加取得**します。
- それまでの間は、**CS@10 等の列は空欄**になりますが、ダッシュボード自体は表示されます
  （勝率・KDA・CS/min・ダメージシェア・キル関与率・対面チャンプ等はタイムライン無しでも表示されます）。

## セットアップ

### 1. ライブラリのインストール（初回のみ）
```
pip install -r requirements.txt
```

### 2. APIキー（`config.json`）
`api_key` にあなたのRiot APIキー、`match_count` は 200、`region` は `asia`（JP/KR）に設定済みです。
**Personalキーは24時間で失効**するので、失効していたら[Developer Portal](https://developer.riotgames.com/)で取り直して差し替えてください。

### 3. 選手リスト（`players.csv`）
17名を登録済み（列: `team,role,nickname,name,tagline`）。
`role` に `コーチ` と入れた選手は**コーチ＝ベンチマーク扱い**になり、既定の集計から除外され参考値として扱われます。
TeamBコーチ（OGMR）はRiot ID未確定のため未登録です（判明したら1行追加）。

## 4ページの内容

### ① 大会全体の概要
サマリーカード（総選手数・総試合数・平均勝率/KDA/CS/min）、チーム別サマリー（行クリックで②へ）、
全体チャンピオン使用×勝率（ロール別フィルタ）、ロール別 平均指標比較。

### ② 選手一覧＆比較
**ロール別タブ**で同ロールだけを比較（最重要改善）。チーム絞り込み、コーチ表示トグル、詳細列の展開。
条件付き書式は**ロール内の相対位置**で色付け（上位=緑/下位=赤、デス等は反転）。選手名クリックで③へ。

### ③ 選手詳細（指導の核）
プロフィール＋ロール内順位付きカード、**成長トラッキング**（試合ごとの推移／指標切替／ロール平均線／移動平均線／コーチ平均線）、
レーニング成績（試合別テーブル＋レーダー）、デスのタイミング分布、チャンピオンプール、直近の試合履歴。

### ④ ロール別ベンチマーク
ロール内の全選手をレーダーで重ね比較（コーチ重ね表示可）、指標別ランキング（横棒）、相関散布図（X/Y軸自由選択）。

| ファイル | 内容 |
|---|---|
| `dashboard.html` | 4ページ構成の再設計ダッシュボード（ブラウザで開く） |

## 「更新」の仕組み・実行時間

- 実行のたびに各選手の**最新200試合のリストを取り直し**、新しい試合の「試合データ」と「タイムライン」だけ追加取得します。
- 試合データ・タイムラインは不変なのでキャッシュし、再取得しません（2回目以降は高速）。
- レート制限（20req/秒・100req/2分）は自動順守、`429` は自動リトライ。途中で止まっても再実行で続きから。

### 時間の目安
1試合につき「試合データ＋タイムライン」の2リクエストになります。
- 既存データのタイムライン追加取得（例：約540試合）→ **約20分前後**。
- 17名×200試合をフルに集める場合は最大 約6,800リクエスト → **2〜3時間**程度（キャッシュにより以降は短縮）。

## ファイル構成

```
lol_senseki_tool/
├─ update.bat / update.py   # 収集→集計→ダッシュボードを一括実行
├─ config.json              # APIキー・リージョン・試合数
├─ players.csv              # 対象選手（team,role,nickname,name,tagline）
├─ collect.py               # Step1: 試合データ＋タイムライン収集
├─ analyze.py               # Step2: 集計・CSV・ダッシュボード生成
├─ metrics.py               # 指標算出（@10・デス分布・ダメージシェア等）
├─ dashboard.py             # 4ページHTMLダッシュボード生成
├─ common.py                # 設定・選手読み込み
├─ data/                    # 収集キャッシュ（matches / timelines / *.json）
└─ output/                  # dashboard.html・CSV
```

ダッシュボードだけ作り直す場合は `python analyze.py`（収集済みデータから再生成）。
